    TypeScript的类型系统

    TypeScript给JavaScript增加了一套静态类型系统，通过 TS Compiler 编译为 JS，编译的过程做类型检查。

    它并没有改变 JavaScript 的语法，只是在 JS 的基础上添加了类型语法，所以被叫做 JavaScript 的超集。

    JavaScript 的标准在不断的发展，TypeScript 的类型系统也在不断完善，因为“超集”的设计理念，这两者可以很好的融合在一起，是不会有冲突的。

    在这里引入23个设计模式的分类，它总体分被为三大类：行为型模式、结构型模式、创建型模式。

    简言之，

    • 创建型模式——关注于分离对象的创建和使用

    TypeScript-基础类型

    基本类型

    前言

    如今TypesScript已经越来越火，不管是服务端（Node.js）,还是前端框架（React、Vue、Angular），都有越来越多的项目使用TS进行开发，作为前端程序员，TS已经成为一项必不可少的技能。本文主要介绍一些TS的基础知识，带大家入门TS。

    TypeScript是什么？

    1、TypeScript是由微软开发的以JavaScript为基础的编程语言。 2、它是JavaScript的一个超集，js是ts的子集； 3、TS可以在任何支持JS的平台上执行； 4、TS不能被JS解析器直接执行，需要编译成JS才可以执行； 5、TS在原先JS基础扩展了JS,并新增了许多变量类型，比如元组、枚举类型等等。
